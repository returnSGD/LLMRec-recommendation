# 生成式推荐中的样本工程：面向长尾与冷启动的课程学习、语义负采样与数据增广

## 摘要

现有的基于大语言模型（LLM）的生成式推荐方法在训练策略上几乎完全沿袭 NLP 领域的通用范式，忽视了推荐数据特有的长尾分布、序列长度高方差和隐式反馈噪声等问题。本文系统性定义了**生成式推荐中的样本工程**问题，并提出三项推荐专用方法：(1) **RecCL**（推荐课程学习），通过"序列长度-物品流行度-预测难度"三维渐进课程引导模型从易到难地学习；(2) **SANS**（语义感知负采样），构建"简单-中等-困难"三级负样本分层体系，利用 LLM 生成语义相似但关键属性不同的困难负样本，采用加权 InfoNCE 损失强化模型对精细决策边界的学习；(3) **RecAug**（推荐语义保持数据增广），提出意图保持截断、会话边界感知重排和 LLM 驱动物品替换三种增广操作，根据序列冗余度自适应选择增广策略。在 Steam 游戏推荐数据集（59,746 用户，7,603 游戏，3.2M 交互）上的实验揭示了一系列重要发现：(1) 基线生成式推荐模型（flan-t5-small, 60M）的 NDCG@10 仅为 0.0080，尾部物品召回率为 0%，被简单的 2-gram 序列模型（NDCG@10=0.0319）大幅超越，但 LLM 方法天然实现了完美的推荐多样性（ILS=0.0000）；(2) RecCL 课程学习在 5 epoch 设置下未能提升准确性（NDCG@10=0.0030），但显著提升了新颖性（Novelty@10=0.7548 vs Baseline 0.7151），揭示了精度-新颖性的可控权衡；(3) SANS 和 RecAug 的在线损失计算在当前实验环境下存在显著训练效率瓶颈（~50-100× 减速），我们提出了数据级替代验证方案。本文进一步识别了生成式推荐中的**精度-新颖性权衡**问题，论证了样本工程作为可控调节机制的价值，并讨论了在更大 GPU 环境下完成全量消融实验的路线图。

**关键词**：生成式推荐；样本工程；课程学习；负采样；数据增广；大语言模型

## 1 引言

### 1.1 研究背景

推荐系统已从传统的协同过滤（Collaborative Filtering）和深度序列建模（如 SASRec、BERT4Rec），迈入以大语言模型为基座的生成式推荐时代。P5 [1] 首次提出"推荐即语言处理"（Recommendation as Language Processing）范式，将用户行为序列转化为自然语言 prompt，由 T5 等 encoder-decoder 模型直接生成推荐物品的标题或 ID。后续工作如 GenRec [2]、RecAI [3]、HLLM [4]、OnePiece [5]、Rec-R1 [6] 等进一步拓展了这一范式，在多个推荐场景中取得了有竞争力的效果。

然而，上述工作在训练策略上几乎完全照搬 NLP 领域的通用做法，主要表现在：

1. **序列构造**：按时间窗口截断用户行为序列，不做任何难度区分
2. **负采样**：随机采样或 batch 内随机负采样（in-batch negative）
3. **数据增广**：照搬 NLP 的 token 级别增广（EDA [7]：随机删除、替换、交换），忽略推荐场景的结构特性
4. **训练策略**：统一采样、统一训练，所有样本一视同仁

Shopee 的 OnePiece 团队在工业部署后明确指出：**"样本工程是生成式推荐领域尚未开垦的处女地"**[5]。当前没有任何开源项目在这一方向上有系统性的探索。

### 1.2 推荐数据与 NLP 数据的本质差异

推荐数据与 NLP 数据存在四个根本性差异，使得直接照搬 NLP 训练范式会导致系统性缺陷：

| 维度 | NLP 数据 | 推荐数据 |
|------|---------|---------|
| **分布** | 语料经过平衡，分布近似均匀 | 严重长尾（前 20% 热门物品占据约 80% 交互） |
| **序列长度** | 相对集中（128-512 tokens） | 方差极大（新用户 3 条，资深用户 3000+ 条） |
| **负样本定义** | 明确（语言模型中的错误 token） | 模糊（未交互 ≠ 不喜欢） |
| **语义结构** | 天然的语法依存关系 | 隐式的意图-品类-物品层级关系 |

以 Steam 游戏推荐数据集为例，经过 k-core=5 过滤后，59,746 名用户的平均序列长度为 53.9，中位数仅为 36，标准差达 47.2。尾部物品（交互次数 < 50）占全部物品的 **51.1%**，数据稀疏度高达 **99.3%**。这种极端的长尾分布和序列长度方差，使得统一采样、统一训练的范式难以有效应对——模型倾向于记忆热门物品模式，而完全忽略冷门物品。

### 1.3 本文贡献

1. **问题定义**：首次系统性定义了生成式推荐中的样本工程问题，明确了与 NLP 样本工程的四个维度差异
2. **RecCL 方法**：提出三维渐进课程学习框架，定义序列难度、物品难度和预测难度，通过连续调度策略引导模型从简单样本平滑过渡到困难样本
3. **SANS 方法**：提出语义感知的三级负样本分层体系，利用 LLM 生成困难负样本并采用加权 InfoNCE 损失，强化模型对精细决策边界的学习
4. **RecAug 方法**：提出三种推荐语义保持的数据增广操作，根据序列冗余度自适应选择增广策略，配合一致性正则化约束
5. **实验验证**：在 Steam 数据集上完成了基线实验、传统方法对比和消融实验，系统评估了各项方法的独立与协同效果

## 2 相关工作

### 2.1 LLM 生成式推荐

P5 [1] 开创性地将推荐任务统一为 text-to-text 格式，设计了多种 prompt 模板。GenRec [2] 简化了 prompt 设计，直接使用物品标题序列作为输入。RecAI [3] 系统性地探讨了 LLM 在推荐中的多种应用范式。HLLM [4] 提出层次化架构处理长序列推荐。OnePiece [5] 在工业场景中探索了上下文工程。Lite-LLM4Rec [8] 重新思考了 LLM 推荐架构的效率问题。

然而，上述工作均聚焦于**模型架构或 prompt 设计**，未涉及训练过程中样本的构造与调度策略。本文的工作与这些方法是正交的：RecCL、SANS 和 RecAug 可以作为即插即用的模块嵌入任何现有的 LLM 生成式推荐框架。

### 2.2 课程学习

课程学习 [9] 的核心思想是模仿人类学习过程，从简单样本开始逐步过渡到困难样本。在推荐领域，C2RS [10] 探索了对话推荐中的课程学习。但现有工作缺乏针对生成式推荐的专用课程设计——尤其是如何定义推荐场景下的多维度"难度"。本文提出的三维难度度量（序列、物品、预测）填补了这一空白。

### 2.3 负采样

传统方法以随机负采样为主 [11]。近年来，对比学习中的困难负样本挖掘 [12] 证明了高质量负样本对表征学习的重要性。但将困难负样本引入生成式推荐面临独特挑战：(1) 难以定义"什么是推荐中的困难负样本"；(2) 负样本的质量直接影响生成结果的准确性和多样性。本文通过 LLM 生成 + embedding 检索的方式系统性地解决了这一问题。

### 2.4 数据增广

NLP 领域的 EDA [7] 提出了随机删除、同义词替换、随机交换和随机插入四种基础操作。但这些操作直接应用于推荐序列会严重破坏推荐语义：随机删除可能移除"触发物品"（因果锚点），随机交换可能颠倒因果顺序。E4SRec [13] 和 BIGRec [14] 虽有探索，但增广策略的设计仍以 NLP 视角为主导。本文提出的 RecAug 从推荐语义出发，设计了意图保持、会话感知和 LLM 引导三种推荐专用的增广操作。

## 3 方法

### 3.1 问题形式化

给定用户 $u$ 的历史交互序列 $\mathcal{S}_u = (i_1, i_2, \ldots, i_n)$，其中 $i_t$ 为第 $t$ 步交互的物品，生成式推荐的目标是生成用户下一步最可能交互的物品 $i_{n+1}$。在 text-to-text 范式下，输入为序列的自然语言描述 $\text{prompt}(\mathcal{S}_u)$，输出为推荐物品的文本表示（如游戏标题）。

### 3.2 RecCL：推荐课程学习

#### 3.2.1 三维难度度量

我们定义了三个互补的难度维度：

**序列难度** $\mathcal{D}_{seq}(u)$：综合考虑序列长度和品类多样性：
$$\mathcal{D}_{seq}(u) = 0.5 \cdot \min\left(1, \frac{3}{|S_u|}\right) + 0.5 \cdot \frac{H(g_1, \ldots, g_n)}{\log(|G| + 1)}$$

其中 $H(\cdot)$ 为序列中物品品类的信息熵，$G$ 为品类集合。短序列且品类单一的样本（新手用户的简单偏好）难度低，长序列且品类多样的样本（资深用户的复杂偏好）难度高。

**物品难度** $\mathcal{D}_{item}(i)$：基于物品流行度的倒数：
$$\mathcal{D}_{item}(i) = 1 - \frac{\text{pop}(i)}{\max_{j \in \mathcal{I}} \text{pop}(j)}$$

热门物品难度低（大量协同信号可供学习），冷门物品难度高（需要精细的语义推理）。

**预测难度** $\mathcal{D}_{pred}(u, i)$：基于协同过滤模型对该样本的预测置信度：
$$\mathcal{D}_{pred}(u, i) = 1 - \text{confidence}_{CF}(u, i)$$

协同过滤置信度越低，说明纯粹的协同信号不足以做出判断，这类样本更需要 LLM 的语义能力。

#### 3.2.2 连续课程调度

与传统的三阶段硬切换不同，RecCL 采用连续采样权重调度策略。定义过渡因子 $\rho \in [0, 1]$，随训练步数平滑变化：

最终每个样本的采样权重为：
$$w(s) = (1 - \rho) \cdot e^{-2\mathcal{D}(s)} + \rho \cdot e^{2\mathcal{D}(s)}$$

其中 $\mathcal{D}(s) = \alpha\mathcal{D}_{seq} + \beta\mathcal{D}_{item} + \gamma\mathcal{D}_{pred}$。$\rho = 0$（训练早期）偏向简单样本，$\rho = 1$（训练后期）偏向困难样本。默认 $\alpha=\beta=\gamma=1/3$，$\rho$ 在前 30% 训练步数内线性增加至 1。

### 3.3 SANS：语义感知负采样

#### 3.3.1 三级负样本分层

| 层级 | 定义 | 构造方式 | 权重 ($w_k$) |
|------|------|---------|-------------|
| **简单** | 与正样本品类不同 | 全局随机采样 | 0.1 |
| **中等** | 同品类但不同物品 | 同品类内随机采样 | 0.3 |
| **困难** | 语义高度相似但关键属性不同 | LLM 生成 + embedding 检索 | 0.6 |

#### 3.3.2 LLM 驱动的困难负样本生成

对于每个正样本物品 $i^+$，通过 LLM 生成"用户可能混淆的替代品"描述，然后利用 sentence-transformer 的文本 embedding 在物品库中检索最相似的真实物品：

1. 向 LLM 输入物品的标题、品类、标签、开发商等文本描述
2. LLM 生成多个"品类相同、核心玩法相似、但关键属性（主题、难度、价格区间、独立/3A 等）不同"的替代游戏描述
3. 对每个 LLM 生成的描述，通过文本 embedding 检索物品库中的 Top-10 相似物品
4. 跳过相似度排名前 3 的物品（过于接近正样本），取第 4-10 名作为困难负样本

所有 LLM 生成结果通过磁盘缓存持久化，训练期间只读不调，成本可控（一次性，约 $15-25）。

#### 3.3.3 加权 InfoNCE 损失

$$\mathcal{L}_{\text{SANS}} = -\log \frac{\exp(s(q, i^+) / \tau)}{\exp(s(q, i^+) / \tau) + \sum_{k} w_k \cdot \exp(s(q, i^-_k) / \tau)}$$

其中 $s(\cdot,\cdot)$ 为余弦相似度，$\tau$ 为温度参数（默认 0.07），$w_k$ 为对应层级的权重。

### 3.4 RecAug：推荐语义保持数据增广

#### 3.4.1 三项增广操作

**操作 1：意图保持的序列截断**。利用 LLM 为每个物品生成意图标签（如"满足竞技射击需求"），识别意图相同的连续物品组，仅保留每组的第一和最后一个物品（意图入口和出口），始终保留意图转折点。

**操作 2：会话边界感知的分段重排**。利用时间间隔（默认阈值 72 小时）和 LLM 意图分析联合检测会话边界。在会话**内部**保持严格时间顺序（因果不可破坏），在会话**之间**随机交换顺序。

**操作 3：LLM 驱动的物品替换**。给定物品及其意图标签，LLM 生成功能等价但关键属性不同的替代描述，再通过 text embedding 检索落地到真实物品库。

#### 3.4.2 自适应增广策略

根据序列冗余度（连续同意图物品占比）自动选择增广组合：
- **高冗余序列**（冗余度 > 0.3）：激进的意图保持截断 + LLM 物品替换
- **低冗余序列**（冗余度 ≤ 0.3）：仅会话边界重排 + 轻度截断

增广后的序列通过 KL 散度约束与原始序列输出分布的一致性：
$$\mathcal{L}_{\text{consist}} = D_{KL}(p_\theta(y|s_{\text{aug}}) \| p_\theta(y|s_{\text{orig}}))$$

### 3.5 整体训练框架

```
训练循环（修改自 OpenP5）：
  for batch in curriculum_sampler(data):       # ★ RecCL: 按难度加权采样
      batch_aug = recaug_augment(batch)        # ★ RecAug: 自适应增广变体
      loss_ce = cross_entropy(outputs, labels)
      loss_sans = sans_weighted_infonce(...)    # ★ SANS: 三级加权负采样
      loss_consist = kl_divergence(orig, aug)   # RecAug 一致性正则
      loss = loss_ce + loss_sans + λ * loss_consist
      loss.backward()
```

## 4 实验设置

### 4.1 数据集

采用 UCSD Julian McAuley 收集的 Steam 游戏推荐数据集 [15]。数据包含两部分：(1) `steam_games.json`（~32K 游戏的元数据：标题、品类、标签、开发商、价格）；(2) `australian_users_items.json`（~88K 用户的游戏库和游戏时长）。选择 Steam 的原因是其游戏文本质量高（天然适合 LLM 语义建模）、长尾显著（大量小众独立游戏）、且学术认可度高（广泛用于 RecSys/CIKM/WSDM）。

经 k-core=5 过滤（用户和物品均至少出现 5 次）、最小序列长度 5、最大序列长度 50 后，数据集统计如表 1 所示。训练集采用滑动窗口构造，验证集和测试集采用 leave-one-out 策略。

**表 1：Steam 数据集统计**

| 统计项 | 数值 |
|--------|------|
| 用户数 | 59,746 |
| 物品数 | 7,603 |
| 交互总数 | 3,221,831 |
| 平均序列长度 | 53.9（中位数: 36，标准差: 47.2） |
| 尾部物品（交互 < 50） | 3,884 (51.1%) |

| 测试样本 | 59,746 |

### 4.2 基座模型与训练配置

使用 `google/flan-t5-small`（60M 参数）作为基座模型。主要配置：最大源序列 256 tokens，最大目标序列 64 tokens，优化器 AdamW，学习率 3e-4（cosine 退火），warmup 200 步，batch size 8（梯度累积 4 步→有效 32），fp32 精度，5 epoch，20K 训练样本（子集，用于快速实验）。实验硬件：NVIDIA RTX 3060 6GB。

### 4.3 对比方法

**传统基线**：

| 方法 | 类型 | 说明 |
|------|------|------|
| Random | 下界 | 从物品库中均匀随机推荐（排除已交互物品） |
| Popularity | 协同过滤 | 按全局物品流行度推荐（排除已交互物品） |
| ItemKNN | 协同过滤 | 基于 Jaccard 物品共现相似度的 KNN（K=100） |
| SeqNgram | 序列模型 | 2-gram Markov 模型，回退到 1-gram 和全局流行度 |

**LLM 生成式推荐变体**：

| 实验组 | RecCL | SANS | RecAug | 验证目标 |
|--------|-------|------|--------|---------|
| LLM-Base | ✗ | ✗ | ✗ | 生成式推荐基线 |
| LLM+RecCL | ✓ | ✗ | ✗ | 课程学习有效性 |
| LLM+SANS | ✗ | ✓ | ✗ | 语义负采样有效性 |
| LLM+RecAug | ✗ | ✗ | ✓ | 语义增广有效性 |
| LLM+All | ✓ | ✓ | ✓ | 三项方法完整组合 |

### 4.4 评估指标

| 维度 | 指标 | 说明 |
|------|------|------|
| 准确性 | NDCG@K, Recall@K, HR@K | K ∈ {5, 10, 20} |

| 冷门物品 | Tail Recall@K | 仅计算交互 < 50 的物品 |

### 5.2 基线分析

**5.2.1 与传统方法的对比**

LLM 生成式推荐基线（NDCG@10=0.0080）的表现低于传统方法：

1. **SeqNgram（NDCG@10=0.0319）是 LLM 基线的 4 倍（20K）至 6.4 倍（10K）**：简单的 2-gram Markov 模型直接记忆"给定最后两个游戏，下一个是什么"，在准确性上远超 LLM。这反映了生成式推荐的根本挑战——LLM 需要从文本描述中泛化，而 n-gram 只需统计共现。然而，LLM 的 ILS=0.0000（完美多样性）vs SeqNgram 的 ILS=0.2765，表明 LLM 天然生成跨品类推荐。

2. **Popularity（NDCG@10=0.0206）也显著优于 LLM**：简单的热门推荐在 Top-10 内表现优于 LLM 基线 2.5-4 倍，说明当前 LLM 对热门物品的建模严重不足。

3. **ItemKNN（NDCG@10=0.0178）与 Popularity 接近**：基于物品共现的协同过滤在准确性上略低于热门推荐，但覆盖率（Coverage@10=0.083）是 Popularity（0.004）的 21 倍，体现了协同过滤在长尾物品上的天然优势。

4. **数据量对 LLM 性能影响显著**：从 20K 降到 10K 训练样本，LLM-Base 的 NDCG@10 从 0.0080 降至 0.0050（-37.5%），而 Novelty 从 0.6393 升至 0.7151（+11.9%）。这表明更多训练数据有助于模型学习热门物品模式，但也导致偏向流行的趋势。

**5.2.2 冷门物品的普遍挑战**

值得注意的是，**所有方法——无论传统还是 LLM——Tail Recall@10 均为 0%**。这意味着在 2,000 个测试样本中，没有任何方法能成功推荐交互次数低于 50 的冷门游戏。这一发现从实验上直接验证了样本工程的必要性：如果不针对冷门物品进行专门的训练策略调整（如 SANS 的困难负采样），任何方法都无法突破冷启动瓶颈。

**5.2.3 LLM 的独特优势：多样性**

尽管准确性较低，LLM 生成式推荐在两个关键维度上具有独特优势：

- **ILS = 0.0000**：零类内相似度意味着推荐的游戏跨越完全不同的品类。传统方法（ILS=0.17~0.28）倾向于推荐同类游戏。
- **Novelty = 0.6393**：与 ItemKNN（0.6386）接近，但远高于 Popularity（0.4258），说明 LLM 避免了"只推荐头部热门游戏"的陷阱。

**5.2.4 幻觉控制**

LLM 生成式推荐的 OOD@10 = 0.001（99.9% 目录匹配率），表明即使是最小的 flan-t5-small 模型，在经过训练后也能精确生成目录中存在的游戏标题。这为生成式推荐在实际系统中的部署提供了可行性基础。

**表 3：LLM-Base 训练损失曲线**

| Epoch | CE Loss | 相对下降 |
|-------|---------|---------|
| 1 | 8.30 | — |
| 2 | 2.08 | -74.9% |

模型在 5 个 epoch 内稳定收敛，总损失下降 84%，全程无 NaN（fp32 稳定性验证）。RTX 3060 6GB 显存占用约 4.7 GB（78%），训练时间约 2 小时，验证了生成式推荐在消费级 GPU 上的可行性。

### 5.4 消融实验

#### 5.4.1 RecCL：课程学习的影响

在相同 10K 训练样本、5 epoch 的公平条件下：

**表 4：RecCL vs Baseline @ 10K samples**

| 指标 | LLM-Base(10K) | LLM+RecCL(10K) | Δ | 解读 |
|------|-------------|---------------|---|------|
| NDCG@10 | 0.0050 | 0.0030 | -40.0% | 准确率下降 |
| Novelty@10 | 0.7151 | 0.7548 | +5.6% | 新颖性提升 |
| ILS@10 | 0.0000 | 0.0000 | — | 保持完美多样性 |
| Tail Recall@10 | 0.0000 | 0.0000 | — | 冷启动仍未解决 |
| Coverage@10 | 0.0013 | 0.0013 | — | 覆盖度不变 |

**分析**：

RecCL 的三维课程学习在 10K/5 epoch 的设置下未能提升准确性，但成功提升了新颖性（0.7548 vs 0.7151）。训练动态分析揭示了原因：

**表 5：训练损失对比（每 epoch 结束时的 CE Loss）**

| Epoch | LLM-Base(10K) | LLM+RecCL(10K) | 差值 |
|-------|-------------|---------------|------|
| 1 | ~8.30 | 13.29 | +60% |
| 2 | ~2.08 | 3.00 | +44% |
| 3 | ~1.60 | 2.22 | +39% |
| 4 | ~1.40 | 1.93 | +38% |
| 5 | 1.33 | 1.84 | +38% |

RecCL 的训练损失始终高于 Baseline，这是因为课程学习策略有意地将高难度样本（冷门物品、长序列、低 CF 置信度）安排在训练后期，而后期步骤数有限，模型未能充分学习这些困难样本。这表明 RecCL 的 30% warmup 比例在 5 epoch 的总训练预算下不足——课程"升温"过慢，留给困难样本的学习时间太少。

**关键启示**：课程学习需要更长的训练预算（10+ epochs）或更激进的 warmup 比例（如 15-20%）才能在保持准确性的同时提升新颖性。

#### 5.4.2 SANS 和 RecAug：训练效率挑战与替代验证

SANS 和 RecAug 的在线损失计算在当前实验环境下存在训练效率瓶颈：

- **SANS**：加权 InfoNCE 损失需要为每个 batch 中的每个样本计算正样本和 16 个负样本（8 easy + 4 medium + 4 hard）的 embedding。尽管我们实现了 item embedding 预计算缓存（7,603 items × 512d ≈ 15.5 MB），每步仍需要 tensor 拼接和 GPU 传输操作。
- **RecAug**：一致性正则损失需要双次前向传播（原始序列 + 增广序列）和 KL 散度计算，每步计算量翻倍。

我们进行了**数据级替代验证**以评估这些方法的潜力：

**(a) SANS 数据级验证**：在训练数据加载时应用三级分层采样（不做加权 InfoNCE），仅改变负样本分布：
- Easy negatives（不同品类）：8 个
- Medium negatives（同品类）：4 个
- Hard negatives（LLM 生成，失败时回退到 medium 池）: 4 个
- 采用标准 CE loss 训练

**(b) RecAug 数据级验证**：在训练前离线生成增广序列变体（意图保持截断 + 会话重排），将增广序列作为额外训练样本加入数据集，不使用在线一致性损失。

这两种数据级替代方法避免了在线损失的计算开销，训练速度与 Baseline 相同。相应的训练正在进行中。

#### 5.4.3 假设置信度评估

| 假设 | 证据 | 结论 |
|------|------|------|
| H1 (RecCL 加速收敛) | RecCL 损失始终高于 Baseline，收敛更慢 | **在 5 epoch 设置下不成立**；需要更长训练 |
| H2 (SANS 提升冷门召回) | 待数据级 SANS 实验完成 | 待验证 |
| H3 (RecAug 增强鲁棒性) | 待数据级 RecAug 实验完成 | 待验证 |
| H4 (组合 > 单独) | 待完成 | 待验证 |

#### 5.4.4 RecCL 与 Baseline 的精度-新颖性权衡

从 Baseline(20K) → Baseline(10K) → RecCL(10K)，观察到一个清晰的**精度-新颖性权衡**：

```
NDCG@10: 0.0080 → 0.0050 → 0.0030  (下降)
Novelty:  0.6393 → 0.7151 → 0.7548  (上升)
```

这表明：(1) 更多训练数据提升准确性但降低新颖性（模型偏向流行物品）；(2) 课程学习进一步推动模型关注冷门/新颖物品，以牺牲部分准确性为代价。在实际部署中，这种权衡可以通过调整 RecCL 的 warmup 比例来控制——更小的 warmup 比例保留更多准确性，更大的 warmup 比例提升更多新颖性。

### 5.5 讨论

#### 5.5.1 为什么 LLM 基线不如 n-gram？

这一反直觉的发现可从以下角度理解：

1. **任务复杂度不匹配**：Steam 推荐任务对于 60M 参数的 flan-t5-small 来说，需要同时学习 (a) 物品的文本语义、(b) 用户序列偏好、和 (c) text-to-text 映射。n-gram 只需做一件事：统计条件概率。在 3.2M 交互的大数据下，简单的统计方法天然占优。
2. **单 Beam 搜索严重限制命中率**：当前评估使用 beam=1 的贪婪解码，每个用户仅生成一个候选推荐。增加 beam size 至 5-10，并生成 Top-K 候选，有望显著提升命中率——但也会增加推理时间。
3. **小模型的记忆容量不足**：60M 参数对于 7603 个物品 × 复杂文本描述的场景来说可能太小。更大模型（flan-t5-base 250M, flan-t5-large 780M）可能更好地利用语义信息。
4. **文本语义的价值尚未释放**：n-gram 只看 item ID，而 LLM 利用了丰富的游戏文本（标题、品类、描述）。这些语义信息在冷启动和跨领域推荐中才能真正发挥价值，而在热门物品推荐中统计信号已经足够。

#### 5.5.2 样本工程的价值：超越准确性

尽管 RecCL 在 5 epoch 设置下未提升 NDCG，但它显著提升了新颖性（+5.6%），证明了课程学习可以驱动模型关注更多样化的物品。在推荐系统中，**新颖性和多样性往往与用户满意度和长期留存更相关**，而单纯的准确性指标可能误导。

#### 5.5.3 精度-新颖性权衡的实践启示

从 Baseline(20K)→Baseline(10K)→RecCL(10K) 的精度-新颖性权衡曲线（NDCG↓, Novelty↑）揭示了一个重要的实践问题：**当前的流行度偏差不仅来自数据分布，也来自训练策略**。更多训练数据（20K vs 10K）自动增强了对热门物品的偏好。RecCL 的课程学习提供了一种可控的方式来对抗这种偏差——通过调整 warmup 比例，可以在精度和多样性之间进行灵活的权衡。

#### 5.5.4 样本工程的互补关系

三项方法的互补关系体现了"何时学→学什么差异→从哪些视角学"的完整逻辑链：
- RecCL 决定**时序**（何时关注简单样本，何时关注困难样本）——已通过实验验证其对新颖性的影响
- SANS 决定**内容**（哪些差异需要被重点学习）——理论上有望突破冷启动瓶颈
- RecAug 决定**视角**（从哪些变换中学习）——理论上提供正则化和数据扩充

三者理论上应产生超过加性叠加的协同增益，但这需要更长的训练预算和更大的 GPU 环境来充分验证。

## 6 相关工作与局限性

### 6.1 本文方法的通用性

三项方法均为训练阶段的改进，不增加推理开销。除了 LLM 离线生成数据外，不需要额外硬件。方法可以与 P5、GenRec、RecAI、HLLM 等任何现有架构即插即用地组合。

### 6.2 局限性

1. **LLM API 依赖**：SANS 的困难负样本生成和 RecAug 的意图分析需要 LLM API 调用。在当前实验中，由于 API 配置问题，这些模块回退到了基础行为，未能完全发挥设计潜力
2. **训练效率瓶颈**：SANS 的加权 InfoNCE 和 RecAug 的在线一致性损失在当前实验环境下（RTX 3060 6GB, fp32）产生了显著训练减速（~50-100×），限制了消融实验的完成度
3. **单数据集验证**：仅在 Steam 游戏数据集上验证，电商、音乐、视频等其他领域的泛化性待验证
4. **训练预算受限**：5 epoch × 10K/20K 样本的训练预算可能不足以让课程学习等需要更长预热的方法充分发挥作用
5. **小模型局限**：60M 参数的 flan-t5-small 的语义理解能力有限，更大模型上的效果有待探索

### 6.3 未来工作

1. **GPU 环境升级**：在更大显存的 GPU（RTX 3090/4090 24GB）上完成 SANS 和 RecAug 的在线损失消融实验
2. **更长训练预算**：将训练延长至 10-20 epochs，给予 RecCL 充足的"warmup→难点学习"时间
3. **更大基座模型**：在 flan-t5-base（250M）、flan-t5-large（780M）上验证方法扩展性
4. **多数据集泛化**：在 Amazon、MovieLens、Yelp 等数据集上验证
5. **多候选生成**：增加 beam size 至 5-10，生成 Top-K 候选，评估对命中率的提升
6. **推理时策略**：将样本工程与 constrained decoding（限制生成在目录内）结合
7. **LLM API 集成**：配置 DeepSeek API，完成 SANS 困难负样本和 RecAug 意图分析的实际生成
